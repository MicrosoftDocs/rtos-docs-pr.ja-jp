# <a name="appendix-c---guix-widget-styles"></a>付録 C - GUIX ウィジェットのスタイル

__"***全般的なスタイル (ほとんどのウィジェットの種類で使用されます):***"__

**GX_STYLE_BORDER_NONE**
  - 値: 0x00000000
  - 説明: このスタイルを使用すると、枠線なしでウィジェットを描画します。

**GX_STYLE_BORDER_RAISED**
  - 値: 0x00000001
  - 説明: 浮き出した枠線でウィジェットを描画します。

**GX_STYLE_BORDER_RECESSED**
  - 値: 0x00000002
  - 説明: くぼんだ枠線でウィジェットを描画します。

**GX_STYLE_BORDER_THIN**
  - 値: 0x00000004
  - 説明: 1 ピクセル幅の枠線を描画します。

**GX_STYLE_BORDER_THICK** 
  - 値: 0x00000008
  - 説明: 太い枠線でウィジェットを描画します。

**GX_STYLE_BORDER_MASK**
  - 値: 0x0000000f
  - 説明: ウィジェット スタイルのメンバーのスタイル フィールドのみをテストするために使用するマスク値です。

**GX_STYLE_TRANSPARENT**
  - 値: 0x10000000
  - 説明: 少なくとも部分的に透明なウィジェットを作成します。 このスタイルは、ウィジェット自体が完全に不透明には描画されない場合に使用します。これには、ウィジェットの背景として半透明なピクセルマップを描画するウィジェットが含まれます。 このスタイル フラグを使用すると、ウィジェットの背景領域を更新するためにウィジェットの親を描画する必要があることが GUIX に伝えられます。

**GX_STYLE_DRAW_SELECTED**
  - 値: 0x20000000
  - 説明: 選択した状態の色とフォントを使用してウィジェットを描画するように指定します。 異なるウィジェットの種類では、異なる方法で DRAW_SELECTED を使用して、ウィジェットが現在選択されていることを示します。

**GX_STYLE_ENABLED**
  - 値: 0x40000000
  - 説明: ウィジェットを有効としてマークします。これにより、ウィジェットによるユーザー入力イベントの受け入れと出力シグナルの生成が可能になります。
  
**GX_STYLE_DYNAMICALLY_ALLOCATED**
  - 値: 0x80000000
  - 説明: ウィジェットの作成時に、gx_system_memory_allocator サービスを使用して、ウィジェット コントロール ブロック メモリを動的に割り当てることを示します。ウィジェットが破棄されるとコントロール ブロック メモリは解放されます。

**GX_STYLE_USE_LOCAL_ALPHA**
  - 値: 0x01000000
  - 説明: ウィジェットを描画するときにローカルのウィジェット アルファ値を使用するように、GUIX の描画関数に指示します。 通常、このフラグは、ウィジェットのフェード アニメーションを実装するために GUIX の内部ロジックによって使用されます。


__"***テキストの配置スタイル (テキストを描画するすべてのウィジェットに適用されるスタイル):***"__

**GX_STYLE_TEXT_LEFT**
  - 値: 0x00001000
  - 説明: テキストが、ウィジェットのクライアント領域内に、左揃えで描画されます。

**GX_STYLE_TEXT_RIGHT** 
  - 値: 0x00002000
  - 説明: テキストが右揃えで、ウィジェットのクライアント領域に描画されます。

**GX_STYLE_TEXT_CENTER**
  - 値: 0x00004000
  - 説明: テキストが中央揃えで、ウィジェットのクライアント領域に描画されます。

**GX_STYLE_TEXT_COPY**
  - 値: 0x00008000
  - 説明: 既定では、テキストを描画するウィジェットには、アプリケーションによって渡されたテキストへのポインターのみが保持されます。 文字列テーブル内で定義されている静的に定義されたテキストの場合、ウィジェットでは、割り当てられたテキストのプライベート コピーを作成する理由はありません。 ただし、ウィジェットに割り当てられるテキストが、sprint() や gx_utility_ltoa などの関数を使用して動的に作成される場合は、割り当てられたテキストの独自のプライベート コピーを保持するようにウィジェットに指示すると便利なことがよくあります。 これにより、アプリケーションでは、テキスト文字列を定義するときに自動変数または一時変数を使用できます。そうでない場合、アプリケーションでは、動的に定義されたテキストを使用するテキスト ウィジェットごとに、静的に定義された文字配列を定義する必要があります。 このスタイル フラグが設定されている場合、ウィジェットでは gx_system_memory_allocator 関数を使用して、割り当てられた文字列のプライベート コピーを保持するために必要なメモリ ブロックが動的に割り当てられます。 したがって、このスタイル フラグの使用は、memory_allocator および memory_deallocator 関数を定義するアプリケーションを前提としています。 GX_STYLE_TEXT_COPY を設定した後は、クリアしないでください。クリアすると予期しない結果が発生します。

__"***ボタン スタイル (GUIX のボタン ウィジェットの種類にのみ適用されます):***"__

**GX_STYLE_BUTTON_PUSHED**
  - 値 0x00000010
  - 説明: ボタンが押された状態、または選択された状態であることを示します。

**GX_STYLE_BUTTON_TOGGLE**
  - 値 0x00000020
  - 説明: クリック イベントが発生するたびに、ボタンの状態が押された状態と押されていない状態で切り替わります。 通常、このスタイルは "チェックボックス" スタイルのボタンと共に使用されます。

**GX_STYLE_BUTTON_RADIO**
  - 値 0x00000040
  - 説明: このスタイルは、ボタンが排他的になり、選択されるとすべての兄弟ボタンが選択解除されることを示します。 通常、このスタイルは "ラジオ ボタン" スタイルのボタンと共に使用されます。

**GX_STYLE_BUTTON_EVENT_ON_PUSH**
  - 値: 0x00000080
  - 説明: 最初に押されたときに、ボタンによってクリック イベントが生成されることを示します。 既定の操作では、ボタンが離されたときにクリック イベントが生成されます。

**GX_STYLE_BUTTON_REPEAT**
  - 値 0x00000100
  - 説明: ボタンが押された状態のままであるときに、ボタンからボタンの親に繰り返しクリック イベントを送信する必要があることを示します。

__"***リスト スタイル (GUIX のリスト ウィジェットの種類にのみ適用されます):***"__

**GX_STYLE_CENTER_SELECTED** 
  - 値: 0x00000010
  - 説明: 予約済み

**GX_STYLE_WRAP**
  - 値 0x00000020
  - 説明: リストが、先頭または末尾のリスト インデックスを超えてドラッグまたはスクロールされると、リストの子が先頭から末尾に折り返されます。

**GX_STYLE_FLICKABLE**
  - 値: 0x00000040
  - 説明: 予約済み

__"***ピクセルマップ ボタンとアイコン ボタンのスタイル:***"__

**GX_STYLE_HALIGN_CENTER**
  - 値: 0x00010000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、水平軸上で中央揃えになります。

**GX_STYLE_HALIGN_LEFT**
  - 値: 0x00020000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、水平軸上で左揃えになります。

**GX_STYLE_HALIGN_RIGHT**
  - 値 0x00040000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、水平軸上で右揃えになります。

**GX_STYLE_VALIGN_CENTER**
  - 値 0x00080000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、垂直軸上で中央揃えになります。

**GX_STYLE_VALIGN_TOP**
  - 値: 0x00100000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、垂直軸上で上端揃えになります。

**GX_STYLE_VALIGN_BOTTOM**
  - 値: 0x00200000
  - 説明: ボタンのピクセルマップは、ボタンの境界内で、垂直水平軸上で下端揃えになります。

__"***スライダー スタイル (GX_SLIDER と派生ウィジェットの種類にのみ適用されます):***"__

**GX_STYLE_SHOW_NEEDLE**
  - 値: 0x00000200
  - 説明: スライダーで針インジケーターを描画するには、このスタイルを含める必要があります。 アプリケーションでスライダーの針を無効にしたり、カスタムの針インジケーターを描画したりする場合は、このスタイルを無効にすることができます。

**GX_STYLE_SHOW_TICKMARKS**
  - 値: 0x00000400
  - 説明: このスタイルが有効になっている場合、スライダー ウィジェットでは目盛りの破線がソフトウェアによって描画されます。

**GX_STYLE_SLIDER_VERTICAL**
  - 値 0x00000800
  - 説明: このスタイル フラグを設定すると垂直スライダーを作成し、このスタイル フラグをクリアすると水平スライダーを作成します。

__"***スプライト スタイル (GX_SPRITE のウィジェットの種類にのみ適用されます):***"__

**GX_STYLE_SPRITE_AUTO**
  - 値: 0x00000010
  - 説明: スプライト ウィジェットが GX_EVENT_SHOW イベントを受信したときに、スプライト アニメーションが自動的に実行されることを示します。

**GX_STYLE_SPRITE_LOOP**
  - 値: 0x00000020
  - 説明: このスタイルを使用する場合、スプライト ウィジェットでは、アプリケーションによってスプライトが停止されるまで、スプライト アニメーション フレームが連続してループします。

__"***ピクセルマップ スライダー スタイル:***"__

**GX_STYLE_TILE_BACKGROUND**
  - 値 0x00001000
  - 説明: スライダーの背景画像を、スプライトの四角形領域全体に並べて表示します。 これにより、小さな縦または横のストライプ画像を使用して、スライダーの背景を埋めることができます。

__"***追加の進行状況バー スタイル:***"__

**GX_STYLE_PROGRESS_PERCENT**
  - 値: 0x00000010
  - 説明: このスタイルを設定すると、進行状況バーのバーの値が、生の値ではなくパーセンテージとして描画されます。 テキストは、進行状況バーの四角形領域内の中央に表示されます。

**GX_STYLE_PROGRESS_TEXT_DRAW**
  - 値: 0x00000020
  - 説明: 現在の進行状況バーの値を、進行状況バー内の中央に表示される 10 進数のテキストとして描画します。

**GX_STYLE_PROGRESS_VERTICAL**
  - 値: 0x0000040
  - 説明: 進行状況が垂直方向になることを示します。 既定値は水平方向です。

**GX_STYLE_PROGRESS_SEGMENT_FILL**:
  - **値**: 0x00000100
  - 説明: 進行状況バーの値は、純色の塗りつぶしではなく、セグメント化された塗りつぶし四角形で示されます。

__"***追加の放射状進行状況バー スタイル:***"__

**GX_STYLE_RADIAL_PROGRESS_ALIAS**
  - 値: 0x00000200
  - 説明: アンチエイリアス ブラシ スタイルを使用して放射状の進行状況バーを描画します。 これにはより多くの CPU 帯域幅が必要になりますが、見栄えもよくなります。 パフォーマンスの低い CPU ターゲットの場合は、このスタイル フラグをクリアすると描画速度が速くなります。

**GX_STYLE_RADIAL_PROGRESS_ROUND**
  - 値: 0x00000400
  - 説明: 放射状進行状況バーの円弧を描画するときに、線の先端が丸いブラシ スタイルを使用します。既定値では線の先端が四角くなります。

__"***追加のテキスト入力スタイル:***"__

**GX_STYLE_ CURSOR_BLINK**
  - 値: 0x00000040
  - 説明: テキスト入力ウィジェットのカーソルは、固定されるのではなく点滅します。

**GX_STYLE_ CURSOR_ALWAYS**
  - 値: 0x00000080
  - 説明 通常、テキスト入力ウィジェットのカーソルは、そのウィジェットに入力フォーカスがある場合にのみ表示されます。 このスタイル フラグを使用すると、入力フォーカスに関係なく、カーソルが常に表示されるようになります。

**GX_STYLE_TEXT_INPUT_NOTIFY_ALL**
  - 値: 0x00000100
  - 説明: このスタイル フラグを使用すると、テキスト入力ウィジェットがキーを押すイベントを受信するたびに、GX_EVENT_TEXT_EDITED イベントが設定されます。

__"***追加のウィンドウ スタイル:***"__

**GX_STYLE_TILE_WALLPAPER**
  - 値: 0x00040000
  - 説明: ウィンドウでは、割り当てられた壁紙画像を、ウィンドウ クライアントの四角形全体に並べて表示します。

**GX_STYLE_AUTO_HSCROLL**
  - 値: 0x00100000
  - 説明: 今後の使用のために予約されています。

**GX_STYLE_AUTO_VSCROLL**
  - 値: 0x00200000
  - 説明: 今後の使用のために予約されています。

__"***追加のメニュー スタイル:***"__

**GX_STYLE_MENU_EXPANDED**
  - 値: 0x00000010
  - 説明: アコーディオン メニュー ウィジェットが、最初に展開された状態になります。

__"***追加のツリー ビュー スタイル:***"__

**GX_STYLE_TREE_VIEW_SHOW_ROOT_LINES**
  - 値: 0x00000010
  - 説明: ツリー ビュー ウィジェットでは、ノード アイコンからルート ツリーノードに線が描画されます。

__"***追加のスクロール バー スタイル:***"__

**GX_SCROLLBAR_BACKGROUND_TILE**
  - 値: 0x00010000
  - 説明: 今後の使用のために予約されています。

**GX_SCROLLBAR_RELATIVE_THUMB**
  - 値: 0x00020000
  - 説明: スクロール バーのつまみの幅 (水平スクロール バーの場合) または高さ (垂直スクロール バーの場合) は、スクロール バーの最小および最大範囲に対する親ウィンドウの可視領域の割合に基づいて計算されます。

**GX_SCROLLBAR_END_BUTTONS**
  - 値: 0x00040000
  - 説明: スクロール バーでは、スクロール バー領域の各端に、ボタンが自動的に作成されて追加されます。

**GX_SCROLLBAR_VERTICAL** 
  - 値: 0x01000000
  - 説明: スクロール バーは垂直方向になります。

**GX_SCROLLBAR_HORIZONTAL**
  - 値: 0x02000000
  - 説明: スクロール バーは水平方向になります。

__"***テキストのスクロール ホイール スタイル:***"__

**GX_STYLE_TEXT_SCROLL_WHEEL_ROUND**
  - 値: 0x00000200
  - 説明: スクロール ホイールでは、正弦波アルゴリズムを使用して、スクロール ホイールが丸みのある形で表示されるようにします。 このスタイル フラグを使用すると、スクロール ホイール ウィジェットのパフォーマンスに大きなオーバーヘッドが生じる可能性がありますが、ホイールの外観を 3D のリアルなものにすることもできます。
