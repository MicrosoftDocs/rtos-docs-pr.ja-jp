# <a name="chapter-1---introduction-to-azure-rtos-guix"></a>第 1 章 - Azure RTOS GUIX の概要

Azure RTOS GUIX (GUIX) は、ThreadX ベースの埋め込みアプリケーション専用に設計された、グラフィカル インターフェイス フレームワークの高パフォーマンスのリアルタイム実装です。 この章では、GUIX の概要と、そのアプリケーションおよび利点について説明します。

## <a name="guix-feature-overview"></a>GUIX の機能の概要

他の多くの GUI 実装とは異なり、GUIX は汎用性を考慮して、つまり、マイクロコントローラー ベースの小規模なアプリケーションから、強力な RISC および DSP プロセッサを使用するアプリケーションまで容易にスケーリングするように設計されています。 これは、当初はワークステーション環境を対象にしていたが、その後で無理に埋め込み設計に詰め込まれたパブリック ドメインまたはその他の商用実装とは大きく異なります。 GUIX の機能の概要を次に示します。

- ホスト ベースのデザイン ツール GUIX Studio による使いやすさ

- 完全なホストされたプロトタイプ作成のための Win32 GUIX ランタイム環境

- ThreadX でサポートされるほとんどのプロセッサをサポート

- ANSI C のみによる記述

- エンディアン ニュートラル

- 最小で、かつ最速の埋め込み GUI

- 実行時に構成可能 (オブジェクト数、画面サイズなど)

- ディスプレイ ドライバー インターフェイスの記述が容易

- カラー (最大 32 bpp の色深度)、モノクロ、グレースケールのサポート

- UTF8 文字列エンコードと文字列リソースによる多言語のサポート

- 既定の無料フォント、新しいフォントの追加が容易

- さまざまなサイズの複数の描画キャンバスのサポート

- さまざまなサイズと色深度の複数のディスプレイのサポート

- 画面の切り替えのサポート (フェード イン、フェード アウト、スワイプなど)

- タッチ スクリーン、ジェスチャ、仮想キーボードのサポート

- ビットマップ圧縮

- アルファ ブレンドのサポート

- ディザーのサポート

- アンチエイリアシングのサポート

- スキニングとテーマ

- キャンバス ブレンド

- 完全なウィンドウ管理

  - 親/子の関係

  - 動的な作成、削除、サイズ変更、移動
  - イベント処理と描画の分離 
  - Z オーダー
  - クリッピングとビュー

- 豊富な一連のウィジェット

  - さまざまなボタンの種類、スライダー、ダイヤル

  - ドロップダウン リスト
  
  - Prompt

  - 複数行テキスト ビュー
  
  - 単一行および複数行テキスト入力
  
  - 数値およびテキスト スクロール ホイール
  
  - ウィンドウとスクロール バー
  
  - 放射状の進行状況バー
  
  - スプライト

### <a name="ansi-c-source-code"></a>ANSI C ソース コード

GUIX は完全に ANSI C で記述されているため、ANSI C コンパイラと ThreadX がサポートされているほぼすべてのプロセッサ アーキテクチャにすぐに移植できます。 ANSI C で記述されていますが、GUIX ではオブジェクト指向モデルと継承を使用します。

### <a name="not-a-black-box"></a>ブラック ボックスではない

GUIX のほとんどのディストリビューションには、完全な C ソース コードが含まれています。 これにより、多くの商用 GUI 実装で発生する "ブラック ボックス" の問題が解消されます。 GUIX を使用すると、アプリケーション開発者は GUI が何を行っているかを正確に確認することができ、秘密は存在しません。

また、ソース コードが存在すると、アプリケーション固有の変更も可能になります。 お勧めはできませんが、必要になった場合は GUI を変更できると間違いなく役立ちます。 これらの機能は、社内製品またはパブリック ドメイン製品の操作に慣れている開発者には特に有効です。 これらの開発者は、ソース コードが存在することと、それを変更できることを期待しています。 GUIX は、このような開発者のための究極の GUI ソフトウェアです。

## <a name="embedded-gui-applications"></a>埋め込み GUI アプリケーション

埋め込み GUI アプリケーションは、ユーザー インターフェイスの要件があり、かつ携帯電話、通信機器、自動車エンジン、レーザー プリンター、医療機器などの製品の内部に隠されたマイクロプロセッサ上で実行されるアプリケーションです。 このようなアプリケーションでは、ほぼ常にメモリとパフォーマンスに何らかの制約があります。 埋め込み GUI のもう 1 つの特徴は、そのソフトウェアとハードウェアが専用であることです。

### <a name="real-time-gui-software"></a>リアルタイム GUI ソフトウェア

基本的に、ある正確な期間内に処理を実行する必要がある GUI ソフトウェアは "*リアルタイム GUI*" ソフトウェアと呼ばれ、時間の制約が GUI アプリケーションに課される場合、そのアプリケーションはリアルタイム アプリケーションとして分類されます。 埋め込み GUI アプリケーションは、外部の環境とのやり取りがその本来の処理であるため、ほぼ常にリアルタイムです。

## <a name="guix-benefits"></a>GUIX の利点

埋め込みアプリケーションで GUIX を使用する主な利点は、高パフォーマンス、豊富な機能、そして非常に小さなメモリ要件です。 GUIX はまた、高パフォーマンスのマルチタスキング Azure RTOS ThreadX リアルタイム オペレーティング システムとも完全に統合されています。

### <a name="improved-responsiveness"></a>応答性の向上

高パフォーマンスの GUIX 製品を使用すると、アプリケーションは、これまでにない速さで応答できます。 これは、膨大な量のビジュアル情報があるか、またはこのような情報の表示に対する厳密なタイミング要件がある埋め込みアプリケーションにとって特に重要です。

### <a name="software-maintenance"></a>ソフトウェアのメンテナンス

GUIX を使用すると、開発者は、その埋め込みアプリケーションの GUI の側面を容易にパーティション分割できるようになります。 このパーティション分割により、開発プロセス全体が容易になり、将来のソフトウェアのメンテナンスが大幅に強化されます。

### <a name="increased-throughput"></a>スループットの向上

GUIX は、埋め込みアプリケーションに直接転送する、使用可能な最高のパフォーマンスの GUI を提供します。 GUIX アプリケーションは、ユーザー インターフェイス情報を GUIX 以外のアプリケーションより高速に処理できます。

### <a name="processor-isolation"></a>プロセッサの分離

GUIX は、アプリケーションと基になるプロセッサおよびディスプレイ ハードウェアとの間の、プロセッサに依存しない堅牢なインターフェイスを提供します。 これにより、開発者はディスプレイ ハードウェアの問題への対処に余分な時間を費やすのではなく、ユーザー インターフェイスの高レベルの側面に集中できるようになります。

### <a name="ease-of-use"></a>使いやすさ

GUIX は、アプリケーション開発者を念頭に置いて設計されています。 GUIX のアーキテクチャやサービス呼び出しインターフェイスは容易に理解できます。 その結果、GUIX 開発者は、その高度な機能をすばやく使用できます。

### <a name="improve-time-to-market"></a>市場投入までの時間の短縮

GUIX の強力な機能は、ソフトウェア開発プロセスを加速します。 GUIX では、プロセッサやディスプレイ ハードウェアのほとんどの問題を抽象化することにより、これらの問題をアプリケーション ユーザー インターフェイス実装の大部分から排除しています。 この機能と、使いやすさや高度な機能セットとが相まって、市場投入までの時間が短縮されます。

### <a name="protecting-the-software-investment"></a>ソフトウェア投資の保護

GUIX は ANSI C のみで記述され、Azure RTOS ThreadX リアルタイム オペレーティング システムと完全に統合されています。 つまり、GUIX アプリケーションは、ThreadX でサポートされているすべてのプロセッサにすぐに移植できます。 さらに良いことに、まったく新しいプロセッサ アーキテクチャを数週間のうちに ThreadX でサポートできます。 その結果、GUIX の使用によってアプリケーションの移行パスが保証され、当初の開発投資が保護されます。
