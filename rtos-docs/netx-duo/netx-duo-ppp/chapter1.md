## <a name="link-control-protocol-lcp"></a><span data-ttu-id="6629c-164">リンク制御プロトコル (LCP)</span><span class="sxs-lookup"><span data-stu-id="6629c-164">Link Control Protocol (LCP)</span></span>

## <a name="challenge-handshake-authentication-protocol-chap"></a><span data-ttu-id="6629c-174">チャレンジ ハンドシェイク認証プロトコル (CHAP)</span><span class="sxs-lookup"><span data-stu-id="6629c-174">Challenge-Handshake Authentication Protocol (CHAP)</span></span>

<span data-ttu-id="6629c-175">CHAP は、PAP よりも複雑な認証プロトコルです。</span><span class="sxs-lookup"><span data-stu-id="6629c-175">The CHAP is a more complex authentication protocol than PAP.</span></span> <span data-ttu-id="6629c-176">CHAP 認証子は、ピアに名前と値を提供します。</span><span class="sxs-lookup"><span data-stu-id="6629c-176">The CHAP authenticator supplies its peer with a name and a value.</span></span> <span data-ttu-id="6629c-177">ピアでは提供された名前を使用して、2 つのエンティティ間で共有される "シークレット" を見つけます。</span><span class="sxs-lookup"><span data-stu-id="6629c-177">The peer then uses the supplied name to find a shared “secret” between the two entities.</span></span> <span data-ttu-id="6629c-178">その後、ID、値、"シークレット" に対して計算が実行されます。</span><span class="sxs-lookup"><span data-stu-id="6629c-178">A computation is then done over the ID, value, and the “secret.”</span></span> <span data-ttu-id="6629c-179">この計算の結果が応答で返されます。</span><span class="sxs-lookup"><span data-stu-id="6629c-179">The result of this computation is returned in the response.</span></span> <span data-ttu-id="6629c-180">正しい場合、PPP は IPCP ステート マシンに進むことができます。</span><span class="sxs-lookup"><span data-stu-id="6629c-180">If correct, PPP can then proceed to the IPCP state machine.</span></span> <span data-ttu-id="6629c-181">結果が正しくない場合、接続は拒否されます。</span><span class="sxs-lookup"><span data-stu-id="6629c-181">Otherwise, if the result is incorrect, the connection is rejected.</span></span>
